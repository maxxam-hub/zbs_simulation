# Симулятор ЗБС

Python-проект для сравнения вариантов бокового горизонтального ствола.

## Текущая постановка модели

- Изотермический режим (`T = const`).
- Допущение «сухой газ».
- Стационарный расчет для каждого сценария.
- Фиксированное забойное давление на пятке (`Pwf_heel`) для сравнения с вертикальным эталоном.
- Уравнение притока:
  - `P_res^2 - P_sf^2 = A*q + B*q^2`
- Перебор сценариев:
  - Длина горизонтального участка (`100..1500 м`, шаг `100 м`).
  - Радиус кривизны (`8, 40, 120 м`).
  - Тип профиля (`flat`, `ascending`).
  - Глубина башмака НКТ от пятки до забоя (`0..L`, шаг `100 м`).
  - Анизотропия `ae` (`0.1, 0.2, 0.3`).

## Зафиксированные входные параметры

- `P_res = 9.5 МПа`
- `T_res = 46 C`
- `k = 345 мД`
- `h = 40 м`
- Относительная плотность газа `gamma_g = 0.693`
- `gas_composition_mol_frac` (по умолчанию для скв. 2205): `{"CH4": 0.9623, "C2H6": 0.0302}`
- При необходимости состав расширяется до C1..C6, N2, CO2, H2S, He, Ar в этом же словаре
- `КГФ = 0.85 г/м3` (в MVP оставлено допущение сухого газа)
- Номинальный диаметр НКТ `114 мм`
- Текущая глубина спуска НКТ `1567.1 м` (в переборе башмак перемещается по ГС)

## Текущие зависимости PVT

- Псевдокритические параметры:
  - `ppc_method = "composition_linear"`: правило Кея (Инструкция 1980, гл. II.3, формулы II.12-II.13),
  - `ppc_method = "composition_precise"`: уточненный расчет K/J (Инструкция 1980, формула II.14),
  - `ppc_method = "specific_gravity_sutton"`: расчет по относительной плотности газа.
- `Z(P,T)`:
  - `z_method = "dranchuk_abou_kassem"` (по умолчанию),
  - `z_method = "aliev_zotov_two_parameter"` (Инструкция 1980, формула II.21),
  - `z_method = "papay"` (упрощенная альтернатива).

## Как менять комбинацию методик

1. Откройте файл `src/zbs_sim/models.py`.
2. Найдите класс `ReservoirParams`.
3. Для выбора метода псевдокритических параметров установите `ppc_method`:
   - `composition_linear` — быстрее и проще, по аддитивным формулам состава.
   - `composition_precise` — более точный вариант по составу (K/J).
   - `specific_gravity_sutton` — когда нет надежного компонентного состава.
4. Для выбора метода расчета `Z(P,T)` установите `z_method`:
   - `dranchuk_abou_kassem` — базовый рабочий метод для автоматических расчетов.
   - `aliev_zotov_two_parameter` — метод по формуле (II.21) из Инструкции.
   - `papay` — быстрый приближенный метод.
5. При необходимости задайте расширенный состав в `gas_composition_mol_frac` в этом же классе, например:
   - `{"CH4": 0.94, "C2H6": 0.03, "C3H8": 0.01, "N2": 0.01, "CO2": 0.01}`
   - Важно: используйте только точные имена компонентов без алиасов.
   - Допустимые ключи: `CH4`, `C2H6`, `C3H8`, `IC4H10`, `NC4H10`, `IC5H12`, `NC5H12`, `NC6H14`, `N2`, `CO2`, `H2S`, `HE`, `AR`.
6. Сохраните файл и запустите расчет:
   - `python run_simulation.py`
7. Проверьте, что расчет прошел:
   - в консоли появится сводка по топ-сценариям;
   - в папке `outputs/` обновятся `scenario_results.csv` и `scenario_results_ru.csv`.
8. Если хотите сравнить две комбинации методик корректно:
   - сначала запускаете первую комбинацию;
   - переименовываете `outputs/scenario_results_ru.csv`, например в `scenario_results_ru_case1.csv`;
   - меняете методы и запускаете вторую комбинацию;
   - сравниваете два файла по колонкам `Дебит_стм3_сут`, `P_сред_по_ГС_МПа`, `Прирост_к_вертикальной_проц`.

## Быстрый запуск

```bash
python run_simulation.py
```

Опционально (чтобы получать PNG-графики, а не только CSV-данные для них):

```bash
python -m pip install -r requirements.txt
```

## Структура модулей

- `src/zbs_sim/reservoir.py`: физика газа (`Z`, `mu`, `rho`).
- `src/zbs_sim/geometry.py`: геометрия ствола и геометрическое сопротивление.
- `src/zbs_sim/completion.py`: сегментация НКТ/затруб и стволовые потери давления.
- `src/zbs_sim/flow_engine.py`: итерационная связка притока и гидравлики.
- `src/zbs_sim/sensitivity.py`: многовариантный перебор и таблица результатов.
- `src/zbs_sim/visualization.py`: построение графиков и экспорт данных графиков.
